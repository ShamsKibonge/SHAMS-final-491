# Portfolio Notes

## Resume Bullets

- Built S.H.A.M.S., an AI-assisted SOC triage prototype that groups OpenSearch security telemetry into analyst-ready cases using Node.js, React, Suricata-style alerts, and Zeek-derived DNS/HTTP/SSL/notice/connection fields.
- Implemented case grouping, fingerprinting, prioritization, and pivot workflows for C2, malware, exploit attempts, DNS tunneling, brute force, reconnaissance, web exploitation, and lateral-movement investigation categories.
- Integrated OpenAI-assisted analysis with bounded JSON outputs, confidence scoring, pivot recommendations, reassessment after OpenSearch queries, and analyst review before ticket submission.
- Connected triage output to MantisBT through duplicate checks, evidence-backed ticket creation, ticket sync, and local analyst memory for prior investigations.
- Designed a React dashboard with case review, evidence inspection, AI analysis, pivot results, ticket preview, automated Case Manager v1 runs, and ticket registry workflows.

## LinkedIn Featured Description

S.H.A.M.S. is a cybersecurity portfolio project focused on SOC automation and alert triage. It collects OpenSearch-backed Suricata and Zeek telemetry, groups related logs into investigation cases, uses AI to assist with assessment and pivot selection, and connects analyst-reviewed findings to MantisBT tickets. The project demonstrates practical security-engineering work across telemetry normalization, case management, evidence preservation, ticketing, and analyst workflow design.
